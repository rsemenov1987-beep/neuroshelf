# UC-001: Регистрация нового пользователя

## Актор 
Пользователь (человек с имплантом NeuroLink)

## Цель 
Создать учетную запись в системе NeuroShelf для доступа к навыкам.

## Предусловия

Пользователь имеет имплант NeuroLink.  
Пользователь не аутентифицирован в системе.

## Основной поток (успешная регистрация):

1. Пользователь заходит на сайт NeuroShelf.
2. Пользователь нажимает кнопку "Регистрация".
3. Система отображает форму регистрации с полями:  
- Имя пользователя (username)  
- Email  
- Пароль  
- Подтверждение пароля  
- Серийный номер импланта (или hardware_hash)  
4. Пользователь заполняет все поля и нажимает "Зарегистрироваться".  

5. Система проверяет данные:  
a. Проверяет, что все обязательные поля заполнены.  
b. Проверяет, что пароль и подтверждение пароля совпадают.  
c. Проверяет, что email соответствует формату.  
d. Проверяет, что email и username не заняты.  
e. Проверяет серийный номер (hardware_hash) на соответствие формату и наличие в белом списке (для MVP - статическая проверка). 

6. Система создает запись в таблице users:  
a. Генерирует UUID для id.  
b. Сохраняет username, email, хеш пароля.  
c. Сохраняет hardware_hash (серийный номер).  
d. Устанавливает is_active = true.  
e. Устанавливает created_at = текущая дата.  

7. Система автоматически аутентифицирует пользователя (создает сессию).  
8. Система перенаправляет пользователя на главную страницу.  
9. Пользователь видит сообщение об успешной регистрации.
